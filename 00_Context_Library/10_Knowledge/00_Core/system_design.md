# AI統合型クリエイティブ環境「my-creative-vault」構築レポート

## 1. コンセプト：Headless Obsidian Workflow
従来の「メモアプリ（Notion）で管理し、チャット（Web）で生成する」スタイルから脱却し、**「IDE（Antigravity）で製造し、Obsidianで脳内を可視化する」**というエンジニアリング・スタイルへ移行しました。

- **基本姿勢**: 執筆活動を「ソフトウェア開発」と同義と捉える。
- **特徴**: Obsidianアプリ本体は「書く場所」ではなく、**「思考のつながり（Graph）と完成形（Preview）を確認するモニター」**として位置づける。

## 2. チーム編成（The Trio）
「誰が何をするか」が明確に定義されました。

| エージェント | 役割 (Role) | 具体的な担当領域 |
| :--- | :--- | :--- |
| **Human (あなた)** | Publisher / PM | 企画決定、最終承認、Goサイン。 |
| **Antigravity**<br>(Gemini 3 Pro) | Director & CSO | **メインの作業場所**。<br>戦略策定、執筆、コーディング。行動指針は `CSO_Persona.md` に準拠。 |
| **Claude Code**<br>(Claude Sonnet 4.5) | Ops / Librarian | **裏方の管理人**。<br>Inboxの自動整理、大量ファイルの一括リネーム、Gitトラブルの解決。 |
| **Obsidian**<br>(App) | Viewer / Monitor | **閲覧専用**。<br>グラフによる発想の結合、記事のプレビュー確認、スマホでのメモ入力。 |

## 3. ディレクトリ構造（The Architecture）
AIが迷わずに自律行動できるよう、明確な住所が定義されました。

### 🧠 00_Context_Library (資産)
プロンプトやナレッジを保管する場所。AIは作業前に必ずここを参照します。
- `10_Knowledge/`
    - `00_Core/`: 企業理念、固定ナレッジ。
    - `10_Glossary/`: `[[ ]]` で生まれたキーワード、用語集。
- `20_Instructions/` (レシピ): 「Podcast台本メーカー」などのプロンプトをMarkdown化して保存。
- `30_Formats/` (お皿): JSONスキーマ、記事のテンプレート、スライド雛形。

### 📥 10_Inbox (作業場)
一時的なメモ、書き散らかしたアイデアの投げ込み場所。
- **運用ルール**: 人間はここに散らかし、1日の終わりにClaude Codeが掃除（移動）する。

### 🏭 20_Production (工場)
- `10_Series/`: 「脱・独学論」のような連載企画。`00_Planning.md` で全体整合性を担保。
- `20_Singles/`: `2026/2026-01/` 等の日付ベース管理。単発エピソード用。
- `assets/`: 画像ファイルの保存先（各記事フォルダ内に自動生成）。

## 4. 新しい制作ワークフロー

### Step 1: 企画・指示 (in Antigravity)
- Antigravityにて「`20_Instructions/podcast_partner.md` になりきって、次の放送の壁打ち相手をして」と依頼。
- 過去の `10_Series` や `10_Knowledge` が自動的に参照され、文脈の通った企画が立ち上がる。

### Step 2: 執筆・製造 (in Antigravity)
- 「`30_Formats/podcast_template.md` の型で台本を書いて」と指示。
- 執筆中、Antigravityのエディタ上でリアルタイムに修正・加筆。
- 図解が必要なら「HTML/CSSで比較表を作って」と指示し、コードベースで画像を作成。

### Step 3: 確認・結合 (in Obsidian)
- Obsidianアプリを開き、プレビューモードで仕上がりを確認。
- 気になったキーワードを `[[ ]]` で囲み、グラフビューで過去の思考とのつながり（セレンディピティ）を発見する。

### Step 4: 整理・整頓 (via Claude Code)
- 作業終了時、Claude Codeに「Inboxの中身を適切なフォルダに移動して」と指示し、環境をクリーンに保つ。

## 5. 導入による3つの変革（メリット）
1.  **プロンプトの「資産化」と「バージョン管理」**
    - Gemの中に隠れていた「秘伝のタレ」が、ファイルとして可視化されました。GitHubで管理されるため、改善の履歴が残り、チーム（AI）全員が同じ基準で動けます。
2.  **文脈（Context）の物理的な結合**
    - 「過去に書いたこと」がすべてローカルファイルとして存在するため、AIはあなたの「全著作」を学習データとして扱えます。シリーズを通して矛盾のない、深みのある発信が可能になります。
3.  **「書く」以外の作業の自動化**
    - ファイルの移動、フォーマット変換、Git操作などの「ノイズ」作業は、すべてAIエージェントに丸投げできます。あなたは「何を伝えるか」だけに集中できます。
