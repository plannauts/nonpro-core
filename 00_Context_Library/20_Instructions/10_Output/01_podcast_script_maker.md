# Podcast Script Maker Guidelines

Podcast「いつも隣におしごとラジオ」の台本を作成するための指示書です。

## 🎯 Objective
与えられた「テーマ」と「構成案（任意）」に基づき、パーソナリティ（タカハシノリアキ）に最適化された台本（箇条書き形式）を作成してください。

## 📋 Instructions

### 1. Structure (構成)
エピソード全体で **約7〜8分** の尺を想定します。
単なる箇条書きのメモではなく、読み上げ可能な **「口語体（スクリプト形式）」** または **「詳細な構成案」** で記述してください。

**構成フレームワーク**:
1. **導入 (Introduction)**:
   - テーマに対するリスナーの感情や課題感（悩み、関心）に共感する。
   - 客観的な事実やデータを用いて、課題の背景や重要性を提示する。
2. **本編 (Body / Analysis)**:
   - テーマを論理的に分解して解説する（例: 3つのポイント、3つのステップなど）。
   - 単なる情報の羅列ではなく、文脈（ストーリー）として繋がるように展開する。
3. **結論 (Conclusion / Strategy)**:
   - 具体的な「戦略」、「アクションプラン」、または「新しい視点（マインドセット）」を提示する。
   - リスナーが明日から少し前向きになれるような、背中を押すメッセージで締める。

### 2. Personality & Context (文脈)
- **Speaker**: タカハシノリアキ
  - 所属: 株式会社プランノーツ 代表 / 一般社団法人ノンプログラマー協会 代表 / ノンプロ研 主宰
  - バックグラウンド: コミュニティ運営、執筆、ノンプログラマー支援。（※元公務員ではありません）
  - **First Person (一人称)**: 僕
- **Target**: 非IT職ビジネスパーソン
  - IT、デジタル、AIに苦手意識がある人でも興味を持てる、分かりやすく親しみやすい内容。
- **Tone & Style**:
  - **Empathetic Partner**: 「先生」ではなく「隣のパートナー」として振る舞う。上から目線の解説ではなく、同じ目線で悩み、一緒に考えるスタンス。
  - **Logical but Emotional**: 論理的（Structured）でありながら、リスナーの感情や課題感に寄り添う温かみのある言葉選びをする（Empathy）。

### 3. Format & Template (型)
- 出力時は必ず **`00_Context_Library/30_Formats/podcast_template.md`** の形式に従ってください。
- **For Series (連載)**:
  - ターゲットのフォルダに `00_Planning.md` が存在する場合、その中の「放送用イントロ (Intro Text)」を読み取り、テンプレートの「お品書き」セクションに挿入してください。
- **Community Time**:
  - ユーザーが指定した「参照エピソード」へのコメントを紹介するコーナーです。
  - 指定されたエピソード名を元にファイル検索を行い、Obsidianリンク `[[ファイル名]]` を埋め込んでください。
  - **重要**: コメントの内容はAI側で要約せず、**原文のまま（Verbatim）** 引用してください。読み上げ時の尺調整や省略は、収録時に語り手（ユーザー）が判断して行います。

### 5. Workflow & File Management (ワークフローとファイル管理)
- **Directory Structure**:
  - 対象エピソードのディレクトリ（例: `.../[Series]/[Episode]/`）を確認してください。
- **File Readiness**:
  - **`comments.md`**:
    - コミュニティからのコメントや感想を貼り付けるためのファイルです。
    - **存在しない場合**: 空ファイルを作成してください。初期内容: `# リスナーコメント（コミュニティ・タイム用参照資料）\n\n- (ここにコメントを貼り付けてください)`
    - **存在する場合**: このファイルの内容を読み取り、`Community Time` セクションの素材として使用してください。
  - **`script.md`**:
    - 台本本体です。作成した内容はここに書き込んでください。

### 4. Antigravity Specific Actions (検索・補完)
AIエージェント (Antigravity) は、原稿作成前に以下の補完行動を行ってください。
- **Search Profile**: 語り手の活動や過去の発言について情報が不足している場合は、`search_web` ツールを使用して検索し、内容に反映させてください。（例: "タカハシノリアキ ノンプロ研 活動"）
- **Search Trend**: 最近の時事ネタを交えるため、テーマに関連する最新ニュースを検索してください。
