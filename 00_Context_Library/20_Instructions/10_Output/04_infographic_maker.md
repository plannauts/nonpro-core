# Web記事からインフォグラフィックを作成

## 入力として与えられるもの
- 「インフォグラフィックを作成してください」
- 記事（Markdown）

## 目的
作成したWeb記事をビジュアルで表現するインフォグラフィックを作成します。インフォグラフィックやSNSに投稿する画像、Web記事内に挿入する画像として使用します。

## 指示
- 全体のタイトル画像と、複数のコンテンツ画像を作成ください。
- 各コンテナには「画像をダウンロード」ボタンを設置し、実際にpng形式で画像をダウンロードできるようにしてください。
- ダウンロードした画像のファイル名は日本語にしてください。

### レイアウト・コンテナについて
- 2カラムレイアウトにしてください。
- 各コンテナは16:9で作成してください。
- 各コンテナのサイズは統一してください。
- 画像化して文字がはみ出ないよう、パディングは十分とるよう、行間等調整してください（**縦方向のはみ出し厳禁**）。
- 16:9のアスペクト比を維持しつつ、文字サイズや余白を調整して内容が全て収まるようにしてください。

### デザインについて
- デザインは記事に合わせて工夫してください。
- 絵文字を活用してください。
- 文中で改行するときは文節の区切りで改行してください。
- スマホ画面でも十分な視認性が得られるように文字は大きくしてください（大見出しと小見出しのみで20px以上、説明文は不要な場合も）。
- **禁止事項**: `bg-clip-text` による文字のグラデーション装飾は、html2canvasでのキャプチャ時に塗りつぶされてしまうため使用しないでください。文字色は単色（solid color）を使用してください。

## 実装上の注意点
- **HTML単一ファイル**として動作するように作成してください（TailwindCSS, html2canvasはCDN利用）。
- 画像のダウンロード機能を実装する場合、レイアウトの安定性を最優先してください。
- **html2canvasの実装**: スクロール位置による画像のズレや、z-indexによる文字隠れを防ぐため、以下の「オフスクリーン配置法」を推奨します。
  1. 対象要素を `cloneNode(true)` で複製。
  2. `document.body` 直下のコンテナに配置。
     - スタイル: `position: fixed; top: 0; left: -9999px; z-index: 10000;`
     - **重要**: `z-index` は負の値ではなく、最前面に来るような正の大きな値を指定してください（負の値だと背景色に埋もれる場合があります）。
  3. そのクローンを `html2canvas` でキャプチャ (`scale: 3` 推奨)。
  4. 完了後にクローンを削除。
  単純な `getElementById` でのキャプチャは避けてください。
