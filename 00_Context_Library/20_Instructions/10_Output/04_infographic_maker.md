# Web記事からインフォグラフィックを作成

## 入力として与えられるもの
- 「インフォグラフィックを作成してください」
- 記事（Markdown）

## 目的
作成したWeb記事をビジュアルで表現するインフォグラフィックを作成します。インフォグラフィックやSNSに投稿する画像、Web記事内に挿入する画像として使用します。

## 指示
- 全体のタイトル画像と、複数のコンテンツ画像を作成ください。
- 各コンテナには「画像をダウンロード」ボタンを設置し、実際にpng形式で画像をダウンロードできるようにしてください。
- ダウンロードした画像のファイル名は日本語にしてください。

### レイアウト・コンテナについて
- **2カラムレイアウト（PC表示時）**にしてください（`grid-cols-1 md:grid-cols-2` 推奨）。
- 各コンテナは `aspect-ratio: 16 / 9` かつ `width: 100%` で作成し、レスポンシブに伸縮させてください。
- **縦方向のはみ出し厳禁**: Flexbox (`flex-col`, `justify-center`) を使用し、内容がコンテナ内に垂直に収まるようにしてください。

### テキスト・デザインについて（最重要）
- **レスポンシブな文字サイズ設定**:
  - `text-5xl` などの固定サイズを指定すると、2カラム表示時（幅が狭まった時）に必ずはみ出します。
  - **必ず `text-xl md:text-2xl` や `text-2xl md:text-3xl` のように、画面幅に応じてフォントが小さくなる設定を使用してください。**
- スマホや狭い画面（2カラム時の各カード）でも視認性を保ちつつ、改行崩れを防ぐためです。
- `bg-clip-text` は `html2canvas` で描画されないため使用禁止です。

## 実装上の注意点
- **HTML単一ファイル**として動作するように作成してください（TailwindCSS, html2canvasはCDN利用）。
- **html2canvasの実装 (WYSIWYGの徹底)**:
  - プレビューとダウンロード画像のレイアウトズレ（改行位置の違いなど）を防ぐため、以下のロジックを厳守してください。
  1. 対象要素を `cloneNode(true)` で複製。
  2. コンテナに配置 (`position: fixed; top: 0; left: -9999px; z-index: 10000;`)。
  3. **重要: クローンの幅・高さには `originalElement.offsetWidth / offsetHeight` （現在の表示サイズ）をそのまま適用してください。**
     - 絶対に `1280px` などの固定幅を強制しないでください。テキストの折り返し位置が変わり、レイアウトが崩れます。
  4. 画質は `html2canvas` の `scale: 3` オプションで確保してください。
  5. 完了後にクローンを削除。
