# Event Planning & Production Workflow

イベントの企画立案から、具体的な制作物（Peatixページ、スライド構成）への落とし込みを行うワークフローです。
エージェントは以下のステップに従って、ユーザーと対話しながら企画を具体化・文書化してください。

## 1. 企画要件の定義 (Planning Session)

以下の要素について、ユーザーへのヒアリングまたは提供されたドキュメントからの抽出を行ってください。

*   **基本情報:**
    *   暫定タイトル (Title)
    *   開催時期・日時 (Date/Time)
    *   開催形式 (Format: Online/Offline/Hybrid)
    *   ターゲット (Target Audience)
*   **コア・コンセプト:**
    *   目的・ねらい (Goal)
    *   参加者へのメリット・解決する課題 (Value Proposition)
    *   コンテキスト・背景 (Context)
*   **コンテンツ構成:**
    *   タイムテーブル案 (Timetable)
    *   主なセッション内容 (Sessions)

## 2. 企画書の作成 (Generate Event Plan)

収集した情報を元に、標準フォーマットで企画書を作成してください。
ファイル名の例: `[ProjectDir]/planning/Event_Plan_[EventName].md`

### 企画書フォーマット (Markdown)
```markdown
# [イベント名] 企画書

**ステータス:** Draft / Review / Fixed

## 1. 基本情報
*   **日時:** YYYY/MM/DD HH:MM - HH:MM
*   **形式:** (例: オンライン Zoom / YouTube Live)
*   **対象:** (ターゲット層)
*   **費用:** (無料/有料)

## 2. コンセプトとタイトル
*   **ねらい:** (このイベントを開催する目的、ゴール)
*   **タイトル案:** (キャッチーかつ内容が伝わるもの)
*   **メッセージ:** (参加者に伝えたい想い)

## 3. イベント構成
(タイムテーブル、セッション詳細)
*   part 1: ...
*   part 2: ...

## 4. 告知プラン
*   **社内向け:** (Slack等での告知方針)
*   **社外向け:** (SNS, Note等での告知方針)
```

## 3. 制作物の生成 (Generate Deliverables)

企画書の内容を元に、以下の制作物ドラフトを作成してください。

### A. Peatixイベントページ原稿
*   参照: `/create_peatix_page` ワークフローのテンプレート構造
*   ファイル名例: `[ProjectDir]/planning/Peatix_Draft_[EventName].md`
*   **要件:**
    *   タイトルはSEO（検索キーワード）を意識すること。
    *   カバーテキストは感情やストーリーを重視し、参加意欲を掻き立てること。
    *   主催者プロフィールやフォーム設定案も含めること。

### B. スライド構成案 (Optional)
*   イベントで使用するプレゼンテーションの構成案（チャプター、各スライドのキーメッセージ）。
*   ファイル名例: `[ProjectDir]/planning/Slide_Outline_[EventName].md`

## 4. レビューとネクストアクション
*   作成したファイルへのリンクを提示し、ユーザーにレビューを依頼する。
*   次のステップ（Peatixでのページ作成、画像作成依頼など）を案内する。
